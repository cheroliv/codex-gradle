- Le développement par surveillance devient exécutable via LangGraph4j (routing + pause/reprise)

== Session 003 — Le Vrai MVP : Boucle d'Autonomie

*Date* : 2026-05-07
*Type* : design (priorisation)
*Modele* : deepseek-v4-pro (via opencode)
*Score* : —

*Objectif* :
- Identifier le vrai MVP : la boucle d'autonomie pour le vibecoding
- Prioriser l'écriture des plugins et du SaaS comme première cible

*Livrables* :
- Section "Le Vrai MVP — La Boucle d'Autonomie pour le Vibecoding" dans AUGMENTED_CONTEXT_PIPELINE.adoc
- Premier jalon concret : codex-pdf extrait Applied AI → JSON LDD → pgvector → assembleCompositeContext → opencode lit le contexte augmenté

*Décisions clés* :
- Le vrai MVP n'est pas Edster — c'est la boucle auto-accélératrice contexte → vibecoding → plugin → contexte
- La première cible est l'écriture des plugins et du SaaS, pas la génération de formations
- Premier jalon : 5 tâches, 1 dossier, 1 livre. Quand ça tourne, le vibecoding passe sous testostérone
- Livre Applied AI choisi stratégiquement : c'est le manuel de construction du moteur
- LangChain4j + LangGraph4j = le capital fixe qui permet au pipeline de s'écrire lui-même
- Double mission d'Edster : former des humains ET former des experts IA
- Les experts IA (employés synthétiques) arriveront avant la formation humaine — ordre inversé par nécessité technique
- Tout le reste (library, trainer, reason, Bloom) = raffinement utile mais pas bloquant
