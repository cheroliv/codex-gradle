== Objectif

et les stocker dans pgvector via R2DBC (connexion reactive PostgreSQL).

== Schema pgvector

[source,sql]
----
CREATE TABLE IF NOT EXISTS codex_documents (
    id              BIGSERIAL PRIMARY KEY,
    source_document TEXT NOT NULL,
    chunk_count     INTEGER NOT NULL,
    license         TEXT NOT NULL,
    created_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE TABLE IF NOT EXISTS codex_chunks (
    id              BIGSERIAL PRIMARY KEY,
    document_id     BIGINT NOT NULL REFERENCES codex_documents(id) ON DELETE CASCADE,
    chunk_index     INTEGER NOT NULL,
    chunk_text      TEXT NOT NULL,
    section_path    TEXT NOT NULL,
    heading_level   INTEGER NOT NULL DEFAULT 0,
    embedding       vector(384),
    created_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
----

== Usage

. `./gradlew chunkDocument -PinputMarkdown=foo.md`
. `./gradlew codexIngest -PchunksFile=build/codex/chunks/foo.json`
  (ou configurer `codex { pgvectorHost = "..." }` dans build.gradle.kts)

== Decisions cles

- R2DBC reactive (pas JDBC) ŌĆö aligne avec codebase-gradle qui declare aussi R2DBC
- Schema pgvector dedie codex (pas reuse codebase-gradle VectorStore)
- Table codex_documents porte `sourceDocument` + `license` (pertinent pour chunks livresques)
- Embeddings calcules 1 par 1 (pas de batching ONNX) ŌĆö suffisant pour volumes documentaires actuels
