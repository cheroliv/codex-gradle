== Objectif

Création de la task `chunkDocument` — découpage sémantique par section
(1 chunk par heading Markdown, pas de taille fixe).

== Livrables

=== 1. ChunkDocumentTask.kt (217 lignes)

Data class `DocumentChunk` avec :
- `id` (SHA-256 tronqué sur `sourceDocument:sectionPath` → `chk-xxxxxxxx`)
- `sourceDocument`, `sectionPath`, `headingLevel`
- `content` (heading + contenu jusqu'au prochain heading)
- `codeBlocks` extraits
- `overlapNext` : 2 premières phrases de la section suivante
- `entities`, `license`

Stratégie de chunking :
- Parse headings Markdown (`#` à `######`) comme délimiteurs
- 1 chunk par section avec son contenu complet
- `sectionPath` reconstruit depuis les ancêtres (ex: `Chapitre 1 > Section 1.2`)
- Overlap configurable via `extractFirstTwoSentences`
- Output : JSON array pretty-printé via `kotlinx.serialization`
