== Objectif

JSON LDD (produit par CDX-1b) → DDL + INSERT PostgreSQL → corpus requêtable en SQL LMD.

== Livrables

- `ImportBookSqlTask.kt` (268 lignes) créé dans `buildSrc/src/main/kotlin/dev/cheroliv/codex/tasks/`
  - Parser JSON récursif sans dépendance externe (zéro kotlinx-serialization ou jackson)
  - Génération DDL CREATE TABLE : `book`, `documents`, `paragraphs`
  - Génération INSERT statements avec hiérarchie `parent_id` cohérente
  - IDs séquentiels globaux (correction de bug compteur réinitialisé entre roots)

- Convention plugin `codex.gradle-plugin-conventions.gradle.kts` : nouvelle tâche `importBookSql` groupe `codex`
- Tâche `importBookSql` visible dans `./gradlew tasks --group codex`

== Test bout en bout


SQL validé : hiérarchie `book → documents (parent_id) → paragraphs (doc_id)`, IDs séquentiels continus (1→6 docs, 1→6 paras).

== Décisions

- Parser JSON maison plutôt que kotlinx-serialization : le JSON LDD actuel est basé sur `DocNode` simple, pas le modèle Book/Part/Chapter/Section complet du AUGMENTED_CONTEXT_PIPELINE spec. Le parser maison est suffisant.
- Pas d'exécution JDBC directe (optionnel dans le PROMPT_REPRISE) — le fichier SQL est autonome, exécutable via `psql -f`.
